GitHub Collaboration Workflow

Welcome to the project! To keep our codebase stable and avoid conflicts, please follow this simple checklist when working on the project.

Golden Rule: Never commit or push directly to the main branch. Always use a separate branch for your work.

1. Before You Start Working

Always make sure your local repository is up to date before making any new changes.

  • Switch to the main branch: 
    git checkout main
  • Pull the latest changes from the remote repository: 
    git pull origin main

2. Creating a Branch

Create a new branch for the specific feature or bug you are working on.

  • Create and switch to a new branch: 
    git checkout -b your-branch-name

(Tip: Use descriptive names like feature-login, bugfix-header, or update-terms)

3. The Pre-Commit Checklist

Before you commit your changes, review this checklist:

  • Tested locally? Have I tested my changes on my local machine?
  • No breaking errors? Does the application still run properly without new errors?
  • Focused changes? Did I only modify the files necessary for this specific feature/fix?
  • Clean code? Did I remove any temporary debugging code (like console.log or var_dump)?

4. Committing Your Changes

Once your checklist is complete, save your work with a clear, descriptive message.

  • Stage your changes: 
    git add .
  • Commit your changes: 
    git commit -m "Briefly describe what you changed"

5. Pushing Your Branch

Push your newly created branch to GitHub so others can see and review it.

  • Push the branch: 
    git push origin your-branch-name

6. Merging into Main (Pull Requests)

Once your branch is pushed, it needs to be merged into main using a Pull Request.

  1. Go to the project's GitHub repository page in your web browser.
  2. You will see a green prompt to "Compare & pull request" for your newly pushed branch. Click it.
  3. Fill out the Pull Request details, describing what you changed.
  4. Wait for review: Ask another collaborator to review your Pull Request.
  5. Merge: Once approved, click the "Merge pull request" button on GitHub to safely merge your code into main.
